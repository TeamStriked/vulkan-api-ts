Parameters object When instantiated with an object, members can be filled directly. Properties VkCmdProcessCommandsInfoNVX.prototype.sType VkStructureType is the type of this structure. VkCmdProcessCommandsInfoNVX.prototype.pNext null is null or a reference to an extension-specific structure. VkCmdProcessCommandsInfoNVX.prototype.objectTable VkObjectTableNVX is the VkObjectTableNVX to be used for the generation process. Only registered objects at the time vkCmdReserveSpaceForCommandsNVX is called, will be taken into account for the reservation. VkCmdProcessCommandsInfoNVX.prototype.indirectCommandsLayout VkIndirectCommandsLayoutNVX is the VkIndirectCommandsLayoutNVX that provides the command sequence to generate. VkCmdProcessCommandsInfoNVX.prototype.indirectCommandsTokenCount Number defines the number of input tokens used. VkCmdProcessCommandsInfoNVX.prototype.pIndirectCommandsTokens Array[VkIndirectCommandsTokenNVX] provides an array of VkIndirectCommandsTokenNVX that reference the input data for each token command. VkCmdProcessCommandsInfoNVX.prototype.maxSequencesCount Number is the maximum number of sequences for which command buffer space will be reserved. If sequencesCountBuffer is VK_NULL_HANDLE, this is also the actual number of sequences generated. VkCmdProcessCommandsInfoNVX.prototype.targetCommandBuffer VkCommandBuffer can be the secondary VkCommandBuffer in which the commands should be recorded. If targetCommandBuffer is null an implicit reservation as well as execution takes place on the processing VkCommandBuffer. VkCmdProcessCommandsInfoNVX.prototype.sequencesCountBuffer VkBuffer can be VkBuffer from which the actual amount of sequences is sourced from as uint32_t value. VkCmdProcessCommandsInfoNVX.prototype.sequencesCountOffset BigInt Number is the byte offset into sequencesCountBuffer where the count value is stored. VkCmdProcessCommandsInfoNVX.prototype.sequencesIndexBuffer VkBuffer must be set if indirectCommandsLayout's VK_INDIRECT_COMMANDS_LAYOUT_USAGE_INDEXED_SEQUENCES_BIT_NVX is set and provides the used sequence indices as uint32_t array. Otherwise it must be VK_NULL_HANDLE. VkCmdProcessCommandsInfoNVX.prototype.sequencesIndexOffset BigInt Number is the byte offset into sequencesIndexBuffer where the index values start. Default Properties VkCmdProcessCommandsInfoNVX.prototype.memoryBuffer ArrayBuffer Native memory reference of the structure.
