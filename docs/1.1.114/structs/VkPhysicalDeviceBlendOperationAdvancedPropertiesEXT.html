Properties VkPhysicalDeviceBlendOperationAdvancedPropertiesEXT.prototype.sType VkStructureType is the type of this structure. VkPhysicalDeviceBlendOperationAdvancedPropertiesEXT.prototype.pNext null is null or a reference to an extension-specific structure. VkPhysicalDeviceBlendOperationAdvancedPropertiesEXT.prototype.advancedBlendMaxColorAttachments Number is one greater than the highest color attachment index that can be used in a subpass, for a pipeline that uses an <>. VkPhysicalDeviceBlendOperationAdvancedPropertiesEXT.prototype.advancedBlendIndependentBlend Boolean specifies whether advanced blend operations can vary per-attachment. VkPhysicalDeviceBlendOperationAdvancedPropertiesEXT.prototype.advancedBlendNonPremultipliedSrcColor Boolean specifies whether the source color can be treated as non-premultiplied. If this is VK_FALSE, then VkPipelineColorBlendAdvancedStateCreateInfoEXT::srcPremultiplied must be VK_TRUE. VkPhysicalDeviceBlendOperationAdvancedPropertiesEXT.prototype.advancedBlendNonPremultipliedDstColor Boolean specifies whether the destination color can be treated as non-premultiplied. If this is VK_FALSE, then VkPipelineColorBlendAdvancedStateCreateInfoEXT::dstPremultiplied must be VK_TRUE. VkPhysicalDeviceBlendOperationAdvancedPropertiesEXT.prototype.advancedBlendCorrelatedOverlap Boolean specifies whether the overlap mode can be treated as correlated. If this is VK_FALSE, then VkPipelineColorBlendAdvancedStateCreateInfoEXT::blendOverlap must be VK_BLEND_OVERLAP_UNCORRELATED_EXT. VkPhysicalDeviceBlendOperationAdvancedPropertiesEXT.prototype.advancedBlendAllOperations Boolean specifies whether all advanced blend operation enums are supported. See the valid usage of VkPipelineColorBlendAttachmentState.
